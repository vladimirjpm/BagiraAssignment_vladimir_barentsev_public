Dear candidate,
As part of the recruitment process at Bagira Systems, you are required to complete this assignment. You will receive the assignment at a specified time (e.g., 9 AM) and must submit your solution within 24 hours (e.g., by 9 AM the next day). Please read the description carefully before starting.

Objectives:
Implement missing parts in a provided code skeleton (backend and frontend) for a small web application used to create training scenarios and manage entities inside each scenario

Instruction:
1. Clone this repository – Bagira-Assignment and create a fork with the following structure – 'BagiraAssignment_<YourName>'
2. The Repo contains frontend skeleton for React and Angular implementation – choose one to implement and specify which one you chose.
3. Follow the README instructions on the backend and frontend folders for detailed instructions.
4. Implement the missing functionality marked in the code (TODOs) and ensure the app works end-to-end.

Requirements:

1. Data rules:
• EntityType must be one of the following - Soldier, Tank, Drone, Aircraft, Vehicle, Civilian.
• TaskForce must be one of the following - Friendly, Enemy.
• Latitude must be between -90 and 90.
• Longitude must be between -180 and 180.
• Entities always belong to exactly one Scenario (no orphan entities).

2. Screens:
• Scenario List – view all scenarios, navigate to details, create.
• Create Scenario – the form should contain name and description.
• Scenario Details – view scenario info and manage its entities.
• Create Entity – the form must include:
  o EntityType
  o TaskForce
  o Name/Callsign
  o Latitude
  o Longitude

3. Backend Requirements (ASP.NET Web API):
Note: Repository implementations are already provided (in-memory storage). Entity count calculation is already implemented. Focus on creating the Application layer and implementing proper error handling.

• Implement the Application layer with business logic services
• Implement proper error handling with appropriate HTTP status codes
• Basic validation (most validation is handled by the frontend)
• Endpoints:
  - GET /api/scenarios – list scenarios (entityCount already calculated)
  - GET /api/scenarios/{scenarioId} – get scenario details
  - POST /api/scenarios – create scenario
  - GET /api/scenarios/{scenarioId}/entities – list entities for a scenario
  - POST /api/scenarios/{scenarioId}/entities – create entity under a scenario
  - GET /api/entities/{entityId} – get entity by ID

4. Frontend Requirements:
Mandatory Features:

• Filtering & Sorting:
  - Scenario List: Search/filter by name or description
  - Entities Table: Filter by Entity Type and TaskForce
  - Entities Table: Sort by Name, Type, Latitude, Longitude, Last Updated
  - Implementation: Client-side filtering/sorting

• Search Functionality:
  - Global search bar in navigation
  - Search scenarios by name/description
  - Search entities by name/callsign
  - Highlight search results

• Data Visualization:
  - Map view for entities (using coordinates)
  - Toggle between table view and map view
  - Display entities on map with Type/TaskForce filtering
  - Click entity on map to see details

• UX Polish:
  - Toast notifications for success/error operations
  - Enhanced loading states (skeleton loaders, button spinners)
  - Enhanced empty states with action buttons
  - Real-time form validation feedback

• Validation:
  - Frontend must validate:
    - Latitude must be between -90 and 90
    - Longitude must be between -180 and 180
    - Entity types are fixed: Soldier, Tank, Drone, Aircraft, Vehicle, Civilian
    - TaskForce must be Friendly or Enemy
    - Required fields (name, etc.)

  ii. Configure CORS in backend to allow frontend origin

  iv. Document any required environment variables

6. Data persistency:
